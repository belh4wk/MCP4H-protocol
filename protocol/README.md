# Protocol

This folder is a signpost, not the implementation itself.

The MCP4H protocol currently lives across these canonical locations:

- `spec/` – media type, cue grammar, profiles, schema lineage
- `schemas/` – released extension-aware schemas
- `examples/messages/` – canonical JSON examples
- `sdk-js/` – reference SDK work
- `tests/` – validation and interop checks

Use this repo section to understand one thing clearly:

**The protocol is the language.**
It is not the connector and it is not the judgment layer.
